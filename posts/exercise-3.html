Exercise 3: PostgreSQL Datatype Exercises

Exercise Description

These exercises are designed to help you understand and practice various data types in PostgreSQL.

Exercise 1: Basic Numeric Data Types

Objective: Understand and use basic numeric data types.

CREATE TABLE

CREATE a table named products with the following columns:

  • product_id (a whole number for each product - integer)
  • product_name (a string of characters, up to 100 characters long, for the product's name - varchar(100))
  • list_price (a precise number that can store values up to 10 digits in total, with 2 digits after the decimal point, suitable for currency - numeric(10,2))
  • stock_quantity (a whole number representing the count of items in stock - integer)
  • weight_kg (a single-precision floating-point number, suitable for storing weights with decimal values - real)

SQL Hint:

CREATE TABLE products (
    product_id integer,
    product_name varchar(100),
    list_price numeric(10,2),
    stock_quantity integer,
    weight_kg real
);

INSERT Data

INSERT at least 3 rows of data INTO the products table, ensuring values match the datatype of each column.

Example:

INSERT INTO products (product_id, product_name, list_price, stock_quantity, weight_kg) VALUES
    (1, 'laptop abc', 12500000.50, 15, 2.15),
    (2, 'smartphone xyz', 7890000.00, 30, 0.20),
    (3, 'computer mouse', 250000.75, 100, 0.10);

QUERY Data

  1. SELECT all products WHERE stock_quantity is greater than 20.
  2. Calculate the sum() of list_price of all products.

Exercise 2: Date and Time Data Types

Objective: Become familiar with date and time related data types.

CREATE TABLE

CREATE a table named orders with the following columns:

  • order_id (a unique, automatically incrementing whole number for each order - serial)
  • customer_name (a string of characters, up to 100 characters long, for the customer's name - varchar(100))
  • order_date (a value representing a specific calendar day, without time information - date)
  • estimated_delivery_time (a value representing a specific point in time, including both date and time, without time zone information - timestamp)
  • order_creation_time (a value representing a specific point in time, including both date and time, and also storing time zone information - timestamp with time zone)

SQL Hint:

CREATE TABLE orders (
    order_id serial,
    customer_name varchar(100),
    order_date date,
    estimated_delivery_time timestamp,
    order_creation_time timestamp with time zone
);

INSERT Data

INSERT at least 3 rows of data INTO the orders table.

Example:

INSERT INTO orders (order_id, customer_name, order_date, estimated_delivery_time, order_creation_time) VALUES
    (101, 'nguyen van a', '2024-05-20', '2024-05-25 10:00:00', '2024-05-20 09:30:00+07'),
    (102, 'tran thi b', '2024-05-22', '2024-05-28 14:30:00', '2024-05-22 13:00:00+07'),
    (103, 'le van c', '2024-05-23', '2024-05-29 09:00:00', '2024-05-23 08:45:00+07');

QUERY Data

  1. SELECT all orders placed in May 2024.
  2. Display order_id, customer_name, and only the order_date (without time).

Exercise 3: Boolean and Text/Character Data Types

Objective: Practice using true/false values and variations of text types.

CREATE TABLE

CREATE a table named my_employees with the following columns:

  • employee_id (a unique, automatically incrementing whole number for each employee - serial)
  • full_name (a flexible string of characters of variable length, for the employee's full name - text)
  • email (a string of characters, up to 255 characters long, for the employee's email - varchar(255))
  • is_active (a true or false value, indicating if the employee is currently active - boolean)
  • country_code (a fixed-length string of exactly 2 characters, for the country code - char(2))

SQL Hint:

CREATE TABLE my_employees (
    employee_id serial,
    full_name text,
    email varchar(255),
    is_active boolean,
    country_code char(2)
);

INSERT Data

INSERT at least 3 rows of data INTO the my_employees table. Try INSERTING one employee with is_active SET to FALSE.

Example:

INSERT INTO my_employees (full_name, email, is_active, country_code) VALUES
    ('nguyen huu nghia', 'nghia.nh@example.com', TRUE, 'vn'),
    ('john doe', 'john.doe@example.com', TRUE, 'us'),
    ('jane smith', 'jane.smith@example.com', FALSE, 'uk');

QUERY Data

  1. SELECT all active my_employees (is_active is TRUE).
  2. SELECT full_name and email of all my_employees WHERE country_code is 'vn'.
